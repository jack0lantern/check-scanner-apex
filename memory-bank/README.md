# Memory Bank

Persistent context for AI-assisted development. Read these files at the start of each task to understand the project and continue work effectively.

## Core Files

| File | Purpose |
|------|---------|
| `project_brief.md` | Core requirements, constraints, scope |
| `product_context.md` | Why the project exists, problems solved, UX goals |
| `system_patterns.md` | Architecture, technical decisions, component boundaries |
| `tech_context.md` | Technologies, setup, project structure |
| `active_context.md` | Current focus, recent changes, next steps |
| `progress.md` | What works, what's left, known issues |

## Update Triggers

Update the memory bank when:

- Implementing significant changes (especially `progress.md`, `active_context.md`)
- Discovering new patterns (add to `system_patterns.md`)
- Resolving bugs (create dated devlog in `.cursor/devlog/YYYYMMDD.md` if that directory exists)

## Reference

Based on the [Cursor Memory Bank](https://github.com/vanzan01/cursor-memory-bank) pattern for documentation-driven AI context.
